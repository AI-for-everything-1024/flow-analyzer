# 微服务消息通讯模式规范

定义微服务及系统组件间的数据交换准则。

## 1. 同步通讯 (Restful / OpenFeign)
用于即时响应的事务处理（如配置读取、状态查询）。
*   **适用场景**: 用户管理、Nacos 配置读取、查询单个告警。
*   **规范**: 
    - 接口返回 `Result<T>` 格式（参考脚手架技能）。
    - 必须配置请求超时（默认 1s）和熔断（Sentinel）。

## 2. 异步通讯 (Kafka / RocketMQ)
用于高吞吐量、事件驱动的解耦（如流量解析、数据上报、告警通知）。
*   **适用场景**: 
    - 解析后的 `FlowRecord` -> 摄取 Kafka。
    - DDoS 检测触发 -> 告警 Kafka -> 发送邮件/短信。
*   **规范**: 
    - **Schema 管理**: 推荐使用 Protobuf 或 JSON。
    - **幂等性**: 消费者必须实现幂等处理（如通过 `flowId` 去重）。
    - **分区策略**: 流量数据按 `dstIp` 分区，以保证同一目标的 Flink 窗口聚合在同一节点。

## 3. 分布式事务与一致性
*   **强一致性**: 尽量避免。若必须，优先使用本地消息表模式。
*   **最终一致性**: 通过消息队列的重试机制实现。

## 4. 链路追踪 (Tracing)
*   所有通讯链路（Feign/Kafka）必须传递 `traceId`。
*   接入 SkyWalking 或 Zipkin 进行全链路拓扑分析。
