# 高级部署策略

根据风险和可用性需求选择合适的策略。

## 1. 滚动更新 (Rolling Update)
*   **机制**: 逐步用新版本的实例替换旧版本。
*   **优点**: 零停机时间，资源利用率高。
*   **缺点**: 部署过程中会存在新旧版本共存（版本倾斜）。

## 2. 蓝绿部署 (Blue-Green)
*   **机制**: 运行两个完全相同的环境。通过 DNS 或负载均衡器将流量从旧环境（蓝）瞬间切到新环境（绿）。
*   **优点**: 切换迅速，回滚极快。
*   **缺点**: 部署期间需要双倍的资源。

## 3. 金丝雀发布 (Canary)
*   **机制**: 先将小部分流量（如 5%）路由到新版本。如果稳定，则逐步增加比例。
*   **优点**: 减小 Bug 影响的范围。
*   **缺点**: 需要先进的流量管理工具（如 Istio, Nginx Ingress）。

## 4. 特性开关 (Feature Flags)
*   **机制**: 将部署与发布解耦。通过逻辑开关为特定用户开启新功能。
*   **优点**: 极高的发布控制安全性。
