# 敏感信息与配置管理

在软件交付生命周期中严密保护敏感信息。

## 密钥管理规则
*   **禁止进入 Git**: 严禁将 API 密钥、密码或证书提交到源码库。使用 `.gitignore` 忽略本地 `.env` 文件。
*   **Vaults / KMS**: 使用专业工具（如 HashiCorp Vault, Kubernetes Secrets）管理生产环境凭证。
*   **动态密钥**: 在支持的情况下，优先使用临时、自动轮转的凭证。

## 配置注入方式
1.  **环境变量**: 适用于简单的非敏感配置。
2.  **ConfigMaps / Nacos**: 用于动态的非敏感应用设置。
3.  **加密属性**: 如果敏感信息必须留在配置文件中，请使用加密工具（如 Jasypt）。

## 监控与审计
*   所有对密钥的访问必须有日志记录。
*   一旦怀疑泄露，应立即轮转所有生产环境密钥。
