# 非功能性需求 (NFR) 模板

在流量分析系统中，非功能性需求往往比功能需求更具挑战性。

## 1. 性能指标 (Performance)
*   **吞吐量**: 系统必须能够处理每秒 ${EXPECTED_EPS} 条 Flow 记录。
*   **延迟**: 
    *   解析延迟（从接收到归一化）必须 < ${PARSE_LATENCY}ms。
    *   端到端分析延迟（从接收到告警产生）必须 < ${E2E_LATENCY}s。
*   **并发**: 支持同时处理来自至少 ${ROUTER_COUNT} 台路由器的并发流量。

## 2. 可靠性与可用性 (Reliability)
*   **数据留存**: 
    *   ODS 层原始数据至少保留 3 天。
    *   DWS 聚合数据至少保留 1 年。
*   **SLA**: 核心采集服务可用性 > 99.99%。
*   **容错**: 允许单台 Flink TaskManager 故障而不丢失数据（通过 Checkpoint 保证）。

## 3. 安全与合规 (Security)
*   **敏感数据**: 内部 IP 地址在导出到外部审计系统前必须进行掩码处理。
*   **访问控制**: 仅授权角色可查询 DWD 层明细数据。

## 4. 可观测性需求 (Observability)
*   **监控**: 所有核心处理环节必须暴露 Prometheus 指标。
*   **审计**: 关键配置变更（如 DDoS 阈值调整）必须记录审计日志。
