# 流量分析典型业务场景集

本文档定义了网络流量分析系统中的核心业务模型与典型使用场景。

## 1. Top-N 流量溯源场景
*   **业务目标**: 实时识别当前网络中带宽占用最大的源 IP 或自治域 (AS)。
*   **BDD 场景**:
    - **假如**: 当前出口带宽使用率超过 80%。
    - **当**: 管理员查询“实时 Top 10 源 AS”。
    - **那么**: 系统应在 1s 内返回结果，并支持下钻 (Drill-down) 到具体的源 IP 及接口。

## 2. 跨境/跨省流量监控
*   **业务目标**: 分析流量的地域流向，用于成本结算和安全审计。
*   **BDD 场景**:
    - **假如**: 已补全 Geo 地理位置数据。
    - **当**: 某国家/地区流量在 5 分钟内突增 300%。
    - **那么**: 系统应自动触发“异常地域流量告警”，并标注所属路由器。

## 3. DDoS 攻防模型 (典型攻击)
| 攻击类型 | 业务特征描述 | 核心维度 |
| :--- | :--- | :--- |
| **SYN Flood** | 单目标 IP 接收到海量 TCP 握手请求，且不完成连接。 | `dst_ip`, `tcp_flags=SYN`, `pps` |
| **UDP 反射攻击** | 源端口为固定服务端口（如 53, 123），包体巨大。 | `src_port`, `dst_ip`, `bps` |
| **ICMP 洪泛** | 网络层不可达消息或 Ping 包异常增多。 | `protocol=ICMP`, `pps` |

## 4. 业务 KPI 定义
*   **解析准确率**: 归一化后的字段与原始 Hex 报文意图一致性 > 99.99%。
*   **告警虚警率**: 在正常业务波动下，DDoS 误报次数 < 1 次/周。
