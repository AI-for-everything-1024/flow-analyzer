# 敏感数据加密与传输安全

确保数据在传输中和持久化时的机密性。

## 1. 传输层安全 (TLS)
*   所有外部暴露的 API 必须使用 HTTPS (TLS 1.2+)。
*   内部微服务间通讯推荐使用 mTLS (相互 TLS)，尤其是在零信任网络环境下。

## 2. 持久化层加密 (Encryption at Rest)
*   **密码加密**: 严禁明文存储或简单 MD5。必须使用加盐哈希算法（如 BCrypt 或 Argon2）。
*   **敏感字段**: 
    - 手机号、身份证、密钥等字段在数据库中应以密文存储。
    - 使用 AES-256 对称加密。密钥应通过环境变量或 KMS (密钥管理服务) 动态注入。

## 3. 密钥管理
*   **严禁硬编码**: 任何加密密钥都不得出现在源码中。
*   **定期轮转**: 建议至少每 90 天轮转一次生产环境的主密钥。
*   **脱敏处理**: 参考脚手架技能，在输出日志和前端展示时对敏感字段进行掩码处理。
