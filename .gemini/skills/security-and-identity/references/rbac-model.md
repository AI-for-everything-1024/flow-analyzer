# 基于 RBAC 的权限控制模型

采用基于角色的访问控制 (Role-Based Access Control) 实现灵活的权限管理。

## 1. 核心实体 (Entities)
*   **用户 (User)**: 系统的使用者。
*   **角色 (Role)**: 权限的逻辑集合（如：管理员、分析师、只读用户）。
*   **权限 (Permission/Authority)**: 具体的原子操作（如：`flow:query`, `ddos:threshold:update`）。

## 2. 权限分层
*   **功能级权限**: 控制菜单可见性、按钮点击（前端）。
*   **接口级权限**: 控制 API 访问（后端）。
    - 使用 Spring Security 注解: `@PreAuthorize("hasAuthority('flow:export')")`。
*   **数据级权限**: 控制用户能看到哪些行/列数据（如：A 路由器的管理员只能看到 A 的数据）。

## 3. 设计规范
*   **多对多关系**: 用户与角色、角色与权限均应为多对多关系。
*   **角色继承**: 支持高级角色继承低级角色的权限。
*   **动态刷新**: 用户权限变更后，其当前活跃的 Token 应在下一次刷新时应用新权限，或通过黑名单机制立即失效。
