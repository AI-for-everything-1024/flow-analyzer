# 性能基准与 EPS 目标

定义系统的“合格”标准，作为上线发布的硬门禁。

## 1. 核心性能基准 (Base Benchmarks)
在标准硬件配置（如 8C 16G 节点）下，核心组件应达到的基准。

| 组件 | 指标类型 | 目标值 (每节点) |
| :--- | :--- | :--- |
| **解析器 (Java)** | 每秒处理 Flow 数 (EPS) | > 50,000 EPS |
| **分析引擎 (Flink)** | 窗口聚合处理能力 | > 100,000 EPS |
| **数仓写入 (Doris)** | 吞吐量 | > 200 Mbps (Stream Load) |

## 2. 响应时间 SLA (P99)
*   **实时告警**: 从报文到达系统到告警发出的端到端延迟 < 5 秒。
*   **分析查询**: Doris 常规多维聚合查询响应时间 < 500 毫秒。

## 3. 稳定性测试 (7x24h)
*   系统在 80% 负载下持续运行 24 小时，不应出现 OOM、FD 泄漏或性能持续衰减。
*   内存消耗曲线应在完成 Full GC 后回到稳定基线。

## 4. 基准测试工作流
1.  **环境对标**: 确保压测环境与生产环境配置一致。
2.  **空载测试**: 确定系统背景资源消耗。
3.  **单点压测**: 找出第一个瓶颈点（通常是 Redis 或 Kafka 分区）。
4.  **集群压测**: 验证水平扩展能力 (Scalability)。
