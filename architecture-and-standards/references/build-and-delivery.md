# 构建、交付与 CI/CD 标准

## 构建系统
*   **一致性**: 在所有环境中使用相同版本的构建工具（例如使用 Maven Wrapper `./mvnw`）。
*   **可复现性**: 确保构建是确定性的，不依赖于本地环境状态。
*   **整洁性**: 定期清理构建产物，并严格管理依赖（杜绝未使用或有风险的库）。

## CI/CD 流水线
1.  **提交 (Commit)**: 代码推送到特性分支。
2.  **Lint 与测试**: 执行静态分析和单元测试。
3.  **构建 (Build)**: 创建制品（Jar/Docker 镜像）并存储到制品库。
4.  **集成测试**: 部署到预发环境并执行集成/端到端测试。
5.  **部署 (Deploy)**: 经手动审核或自动触发持续交付到生产环境。

## 版本管理
*   **语义化版本 (SemVer)**: `主版本.次版本.修订号`
    *   `主版本`: 不兼容的 API 变更。
    *   `次版本`: 向下兼容的功能新增。
    *   `修订号`: 向下兼容的问题修复。
*   **Git 工作流**: 使用 `main` 分支存放生产代码，`develop` 分支用于集成，`feature/*` 用于开发。
