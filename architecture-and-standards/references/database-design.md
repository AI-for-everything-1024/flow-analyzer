# 数据库与数据视图设计 (OLTP & OLAP)

标准化数据在不同数据库引擎中的存储、索引和分析方式。

## 1. OLTP 设计标准 (MySQL 8)
用于高并发、事务性处理（如配置、用户元数据）。

*   **数据模型**: 遵循第三范式 (3NF) 以最小化冗余。
*   **引擎**: 严格使用 InnoDB。
*   **字符集**: `utf8mb4` / `utf8mb4_general_ci`。
*   **核心实践**:
    - **主键**: 始终使用 `BIGINT UNSIGNED` 配合 `AUTO_INCREMENT` 或 ULID/Snowflake。
    - **索引**: 为高基数（High-cardinality）列使用 B-Tree 索引。避免 `SELECT *`。
    - **事务**: 保持事务短小以最小化锁竞争。
*   **命名**: 表名和列名使用 `snake_case`。

## 2. OLAP 设计标准 (Apache Doris)
用于大规模分析处理和多维查询。

### Doris 中的存储模型
*   **聚合模型 (Aggregate)**: 用于预聚合汇总（如流量计数）。
*   **唯一模型 (Unique)**: 用于基于主键的更新（如告警状态）。
*   **明细模型 (Duplicate)**: 用于存储允许重复的原始记录。

### 数据仓库分层 (5 层架构)
*   **ODS (原始数据层)**: 原始、不可变的记录（保留 1-3 天）。
*   **DIM (维度层)**: 元数据（IP 地理位置、ASN、路由器信息）。
*   **DWD (明细数据层)**: 与维度关联后的增强记录（用于 24 小时深度取证）。
*   **DWS (服务数据层)**: 预聚合的时间桶汇总（1 分钟/5 分钟）。
*   **ADS (应用数据层)**: 用于仪表盘和告警的最终指标。

## 3. 数据视图选择准则
*   **事务性?** -> MySQL (OLTP)。
*   **多维聚合?** -> Doris (OLAP)。
*   **数亿行数据的高性能关联?** -> Doris (MPP 架构)。
*   **低延迟键值对查找?** -> Redis。
