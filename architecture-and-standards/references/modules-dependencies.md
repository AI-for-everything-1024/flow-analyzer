# 模块化与依赖管理

## 模块分解
*   **高内聚**: 将相关的逻辑保持在同一个模块内。
*   **低耦合**: 最小化模块间的依赖。
*   **垂直 vs 水平**: 优先选择按功能切分的垂直切片，而非严格按技术分层的水平分层，以获得更好的敏捷性。

## 依赖规则
*   **单向流**: 依赖关系应从外层（实现）流向内层（领域）。
*   **禁止循环依赖**: 使用工具（如 SonarQube, ArchUnit）检测并防止循环引用。
*   **基于接口的耦合**: 模块间应通过接口/抽象进行交互，而非具体实现。

## 包结构
*   `api/`: 面向外部的接口/合约。
*   `core/`: 纯业务逻辑 (领域层)。
*   `infra/`: 数据库、外部 API 客户端、消息中间件实现。
*   `app/`: 应用服务/编排层。
