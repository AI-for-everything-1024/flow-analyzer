# 架构度量与审视

通过自动化的架构检查和指标追踪来维护系统完整性。

## 1. 自动化的架构测试 (如 ArchUnit)
以代码形式实现规则，防止架构腐化：
*   **层级违规**: 严禁 `infra` 层代码直接依赖 `app` 或 `core` 层。
*   **循环依赖**: 严禁包或模块之间存在循环引用。
*   **命名一致性**: 所有 `service` 实现类必须以 `ServiceImpl` 结尾。

## 2. 架构审视的关键指标
*   **耦合与内聚**: 监控入向耦合 (Afferent) 和出向耦合 (Efferent)。
*   **不稳定性指数**: 识别那些难以修改且容易引起回归的模块。
*   **认知复杂度**: 使用 SonarQube 追踪复杂的类或方法。如果单个方法的复杂度超过 15 点，则触发告警。

## 3. 定期架构审视清单
*   **4+1 视图是否仍然准确？**
*   **技术债 (TODOs) 是否在增长？**
*   **系统的可扩展性是否仍满足 SLA？**
*   **是否需要将某些单体模块拆分为微服务？**
