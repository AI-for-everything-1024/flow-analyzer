# 代码评审与验证标准

## 1. 自动化代码评审 (自我纠错)
*   **Pre-commit Hooks**: 在 `git push` 前自动运行 Lint 和单元测试。
*   **AI 辅助评审**: 使用 AI Agent 对 Pull Request 进行初步审查：
    - 业务逻辑正确性。
    - 是否符合项目编码标准。
    - 识别遗漏的边界情况。

## 2. 同行评审 (Peer Review)
*   **每个 PR 至少需要 1 人批准。**
*   **评审重点**:
    - **可读性**: 意图是否清晰？
    - **可维护性**: 代码是否易于修改或扩展？
    - **安全性**: 是否存在数据泄漏或漏洞模式？
    - **测试**: 是否包含有意义的测试用例？

## 3. 集成与部署评审
*   **预发验证**: 在类生产环境中执行手动或自动的烟雾测试。
*   **性能基准**: 将当前版本的性能指标（延迟/吞吐量）与上一个稳定版本进行对比。
