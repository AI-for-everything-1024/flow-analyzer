# 环境分级与管理

标准化不同部署环境的用途和隔离级别。

## 环境级别
| 级别 | 用途 | 数据源 | SLA |
| :--- | :--- | :--- | :--- |
| **开发 (Dev)** | 特性开发与本地集成。 | Mock / 本地数据库 | 低 |
| **测试 (QA)** | 系统集成与质量保证。 | 脱敏后的生产数据 | 中 |
| **预发 (Staging)** | 最终验证，与生产环境完全一致。 | 生产数据快照 | 高 |
| **生产 (Prod)** | 服务于真实终端用户。 | 实时生产数据 | 极高 |

## 命名空间与资源隔离
*   **逻辑隔离**: 使用 Kubernetes Namespaces 或 VPC 子网。
*   **权限隔离**: 为每个环境使用独立的 IAM 角色或服务账号。

## 环境特定配置
*   使用 `application-${env}.yaml` 或 Nacos 命名空间。
*   **关键**: 严禁在源代码中硬编码环境特定值。应使用环境变量或集中的配置中心。
