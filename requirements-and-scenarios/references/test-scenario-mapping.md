# 边界情况与测试场景映射

通过前瞻性思考确保系统的健壮性。

## 1. 边界情况清单 (主动防御)
*   **空状态**: 如果列表为空会怎样？如果用户没有数据会怎样？
*   **并发**: 如果两个用户在同一微秒点击“保存”会怎样？
*   **数据限制**: 如果输入字符串长达 10,000 个字符会怎样？如果文件大小为 2GB 会怎样？
*   **权限**: 如果用户手动尝试访问未授权的 URL 会怎样？
*   **网络**: 如果请求中途连接中断会怎样？

## 2. 映射到测试用例
每个用户故事场景都应直接映射到一个或多个测试用例：
*   **正向测试**: 路径正常、万事顺利的场景。
*   **负向测试**: 用户提供无效输入或非法操作的场景。
*   **破坏性测试**: 外部依赖（数据库/API）故障的场景。

## 3. 非功能性场景
*   **性能**: 95% 的请求必须在 200ms 内响应。
*   **可扩展性**: 必须能处理 1,000 个并发用户。
*   **易用性**: 必须符合 WCAG 2.1 AA 级标准。
